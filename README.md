本技术报告旨在分享微调模型的技术细节。
背景：
本人爱好写小说，奈何在语文的一些语法和“的地得”方面不老练，最生涩的时候一篇2000字的文本能找到十处“的地得”错误。
当我寻找有没有哪个模型能够帮我纠错时，发现并未有任何一种模型对其进行专项训练，也很少有见到明确的评估指标与数据。
“的地得”作为中文最常出现的错误之一，却鲜少有关于该任务专项模型与数据的资料。由此，我开始了训练模型的想法，试图用其纠正我的小说错误。
首先，我在ai的帮助下设计了一个bigru模型，从此开始了我的数据收集之路。该模型是从0开始训练，验证集的F1为0.8，但根据我的使用体验和预估水平，现实世界中f1值仅为0.6～0.7。
于是在2026年的5月15号，我有了微调模型的想法。我微调了三个模型，分别为RoBERTa-wwm-ext的110m版本、MacBERT-Large、
RoBERTa-wwm-ext-large，其中RoBERTa-wwm-ext-large表现最佳。

技术细节与报告：
一.训练数据：
训练数据的其中50%来源于我的小说文本、书籍摘抄、聊天口语句子、网络上收集的句子、我人工编写的句子。另外50%来源于开发数据，其中包含人民日报、古诗、文言文等共1万条。
训练数据包含正确的句子、正确对错误的句子。比例较为均衡且贴近真实世界的分布。

二.模型训练：
基于谷歌云服务的T4GPU，全程在手机上观察训练。
1. 任务形式
将句子中所有出现的“的”“地”“得”替换为 [MASK]，其余位置保持不变。模型需在掩码位置上预测正确的原汉字，等同于针对这三个特定 token 的掩码语言模型（MLM）填空任务。

2. 模型架构
基座模型：hfl/chinese-roberta-wwm-ext-large
冻结策略：冻结嵌入层及前16 层Transformer（占比66%），仅微调后续层和 MLM 头，可训练参数量大幅减少。
显存优化：启用梯度检查点（gradient_checkpointing_enable）。

3. 数据处理
最大序列长度：350（截断）
自定义 collate_fn：动态识别输入中的目标 token（的/地/得），将其替换为 [MASK]，标签仅保留目标位置的 token id，其余位置设为 -100，损失仅计算目标位置。
训练/验证数据从 JSON 文件读取。
由于模型使用掩码策略，故训练数据里的正确——错误对被转换为纯正确的句子，遮蔽目标字。

4. 训练策略
配置项 数值/方法
优化器 AdamW
学习率 2.5e-5
权重衰减 0.01
学习率调度 余弦退火 + 线性预热（10% 总步数）
有效批次大小 16（micro batch 8 × 梯度累积 2）
混合精度 torch.amp（GradScaler）
梯度裁剪 max_norm = 1.0
最大训练轮次 7
早停耐心值 3（基于验证 loss）

5. 安全回滚机制
每轮验证结束后：
若验证 loss 创新低，保存当前模型为最优检查点，清零计数器。
若验证 loss 未下降：
计数器 +1；
立即回滚：删除当前模型，重新加载最优检查点，并冻结对应层；
为回滚后的模型重建优化器与调度器，再根据当前轮次快进调度器至对应步数，保证后续训练步数与计划一致；
连续 3 轮未提升则触发早停。
此设计避免模型在验证 loss 上升后继续沿错误方向更新，同时保留利用后续数据重新探索的可能。最终，模型在第三轮达到最低的验证集loss。

6. 评估指标
每轮验证结束后，分别计算 “的”“地”“得”各自的 F1 值（macro 或 binary），同时统计每个类别的真实数量与预测数量，提供直观诊断。训练日志如下：

7. 总结
该方案通过冻结底层、精细化标签处理、混合精度与回滚式早停，在有限参数量下高效微调大模型，专门优化“的/地/得”三个高混淆助词的辨识能力。训练脚本具备完整的复现性与异常保护机制。

三.补充说明
模型在我的手机本地推理，CPU为天玑9400，使用INT8动态量化。采用按句子分割的形式接收文本，实测处理3000字的小说需要25s左右，处理13000字的小说需要200s左右。
在我个人的实际体验中，模型在绝大多数的文本都能正确地揪出错误并修正，准确率预计在90%以上。该模型完全达到了帮我审核小说“的地得”错误的目的，可用性很高。
值得注意的是，模型仍有可能犯一些低级错误，这可能和神经网络的特性或数据有关。通过后处理，模型在现实世界中的准确率还能在上浮几个百分点。
